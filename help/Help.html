Andrego

Chris Caldwell - 12/19/2023

This game was inspired by Fitbit's "Bingo" game, obsoleted by Google in 2022. Andrego was made with no reference to the original Fitbit code, and neither game looks all that much like the game "Bingo" played in church basements everywhere.

Your introduction to Andrego will probably come in the form of an E-mail inviting you to an existing game. That e-mail will include a hyperlink to the program. Follow that link, enter your Andrego id (brightsands.com user id) and password and it should get you to the main screen.

To even get an invite, you already have an id and password, perhaps given to you by the administrator. Eventually, new Andrego players will be able to go to the login screen and create their own id and password, but that is not tested as of this writing.

At the main screen, you'll see several buttons which take you to different menus.

The "Players" screen will give you statistical information for the specified player (number of any task performed, averages etc. If you click on a particular task, you can create a new entry for that task.

Tasks will give you a list of all the tasks Andrego knows about with a push-button which will take you to that task's page.

Account administration allows you to set your password and other information.

Dump user log shows you all the tasks you've ever done.

The interesting button here is "Games" which shows you all the games that you are either participating in or managing. Click "Play" on a particular game to (re)join it.

Tiles that are light blue must be flipped if you are to win the game. White tiles may or may not be useful (some of the white tiles will allow you to make other tiles easier or swap them). Darker blue tiles are required but are now flipped over. Grey tiles were not required but have now been flipped over. Each tile is labeled with the task you have or must perform to flip it. Unflipped tiles also have the number of units you must have to flip the tile.

If you click on a non-required (white) tile and you have enough of whatever task it requires to flip it, it may give you the opportunity to swap two other tiles, or deduct work from another tile. At the bottom of this grid are sections for each task involved in this game. You can click on these sections to log entries for that task. Below that, you can see the status of each players (and how many tiles they have flipped).

Important things to know about Andrego:

  • As of this writing, it does not interface to any health device's API so it can't, for instance, know how many steps your Fitbit is showing without you entering that information.
  • Consequently, you need to enter this information by hand including when these tasks happened.
  • Consequently, there is nothing preventing you from cheating.
But there is a silver lining: Since we're not limited to what APIs we know how to interface to (we don't know how to interface to any!), we can measure anything and everything that can be entered by hand. If you want to create a task counting the number of times you burp in the key of C, go ahead, but just remember, you're going to need to enter the data as you go.

Each Andrego user has a log of all the tasks they have completed going back to the first time they ran Andrego. Based on that log, Andrego calculates how many of each task the user performs each day based on the last 4 days they logged anything for that task (which is to say that if you don't log that task on a particular day, it is simply not used for the average calculation). For instance, consider the last 8 days of an Andrego user's step log:

DaySteps loggedUse for average to calculate goal
1 0 Ignored
2 8000 Ignored
3 5000 Yes
4 0 Ignored
5 6000 Yes
6 6200 Yes
7 0 Ignored
8 7000 Yes
Total used for calculation 24200 /4 = 6050
In this example, 24200 steps over the 4 days steps were logged equals 6050 steps/day. Andrego will typically set the goal of a particular game (that includes that task) to be that number per day. If a game runs for 10 days, it will use 10 times that average etc or lay the game out so that you'll have accomplished 242000 steps in order to win (in the above example).

As a design choice, no Andrego user is able to see any other user's task log or goals, but of course they are free to exchange this information through other media. This might change.

(game name) player A player is an Andrego user with an invitation to play a particular game. For instance, if somebody creates a game called "Popeye" (the only tasks are strength training and eating vegetables), a "Popeye player" is someone who has been invited to that game.
Game A game consists of:
AttributeMeaning
Starting and ending time Entries logged between these times can be used in this game.
Pattern One of several files each of which contains a configuration of tiles that are either required or not required to win the game, e.g. "Diamond", "Smile", etc. The only real limit on the pattern is that it must be rectangular and contain at least one required tile.
Invitees A selection of one or more Andrego users
Task % For every possible task, designate the fraction of the board that will have that task. Percentage will be calcuated by multiplying this value 100 and dividing by the sum of all the task values. If you have 3 tasks that are non-zero: 1, 2 and 1, they will be calculated as 25%, 50%, 25%.
